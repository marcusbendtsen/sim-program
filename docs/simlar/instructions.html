Instructions

All systematic reviews should be registered in PROSPERO and a protocol based on PROSPERO items should be deposited in the sim-program GitHub repository prior to review commencement.

Planning and piloting stage

  1. Clearly define the research question. This typically includes deciding on the antecedent and consequent, e.g., alcohol consumption → liver cancer.
  2. Clearly define the estimand(s) of interest, e.g., relative risk of liver cancer given exposure to grams of alcohol consumed per week.
  3. Define inclusion and exclusion criteria:
    • Language of published reports.
    • Study types, likely to be all quantitative study types excluding case studies.
    • Specific populations that should be included or excluded (age, riskfactors, etc.).
    • Settings (may want to exclude overly specific settings which may induce selction bias).
  4. Decide on the data that should be extracted, e.g., population characteristics, risk estimates, etc.
  5. Create a form that can be used to store extracted data and upload to the sim-program GitHub repository.
  6. Select keyword blocks from the SIM program's pre-specified queries.
  7. Select sources that should be searched. The following should always be included:
    • PubMed
    • Cochrane Central Register of Controlled Trials (CENTRAL)
    • Cochrane Database of Systematic Reviews (CDSR)
    • Web of Science
    • Scopus
    • PsycArticles
    • PsycInfo

Intermediate stage

If a systematic review is found during planning and piloting that is recent and acceptably answers the planned research question, then stop here and consider if a systematic review is necessary at this time or not.

Execution stage

  1. Get a team together consisting of at least 3 people (including the lead).
  2. Search all databases:
    • Create one file for each database (in .bibtex format) and add to sim-prorgram GitHub repository.
    • Create one combined file for all datasets with duplicates removed using Zotero (in .bibtex format) and add to sim-program GitHub repository.
  3. At least two team members screens all titles and abstracts independently of one another.
    • Create one file of pruned articles with each person's results (in .bibtex format) and add to sim-program GitHub repository.
    • Create a combined file of pruned articles with duplicates removed (in .bibtex format) and add to sim-program GitHub repository.
  4. At least two team members reads all full-text articles and decides which to include.
    • Create one file with articles kept per person reading full-text (in .bibtext format) and add to sim-program GitHub repository.
    • Create one file with all articles combined with final decision (in .bibtext format) and add to sim-program GitHub repository.
    • Make notes of exclusion reason at this stage.
    • Make notes of any discussions about inclusion of reports at this stage (disagreements).
    • Find an arbiter if no consensus can be found.

Synthesis and bias assessment

  1. At least two team members use the previously created form to extract data from each report.
    • Compare and discuss differences.
    • Contact authors if necessary.
    • Find an arbiter if no consensus can be found.
    • Make notes of any differences, disagreements, requests from authors, and use of arbiter. Add notes to repository.
  2. Create a flow chart for the selection process.
  3. At least two team members independently assesses the risk of bias using ROBINS or ROB (if RCT).
    • Comment on all judgements and add a file per person to the repository.
    • Combine judgements and make note of any discussions regarding disagreements.
    • Find an arbiter if no consensus can be found.
    • Make notes of any differences, disagreements, requests from authors, and use of arbiter. Add notes to repository.
  4. Creat risk of bias matrices/tables.
  5. Create a summary table showing the characteristics of the studies.
  6. Create a summary table which shows the quantitative estimates of the estimands.
  7. If suitable, conduct a meta-analysis.
  8. Assess publication bias using funnel-plots.
  9. Use GRADE to assess overall quality of evidence for each estimand.
