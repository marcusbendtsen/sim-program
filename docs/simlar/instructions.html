Instructions

Piloting stage

  1. Clearly define the research question. This typically includes deciding on the antecedent and consequent, e.g., alcohol consumption → cancers.
  2. Clearly define the estimand(s) of interest, e.g., relative risk of liver cancer given exposure to grams of alcohol consumed per week.
  3. Select keyword blocks from the SIM program's pre-specified queries (queries).
  4. Pilot the searches and look specifically for systematic reviews and meta-analyses.

Intermediate stage

If a systematic review is found during piloting that is recent and acceptably answers the planned research question, then stop here and consider if a systematic review is necessary at this time or not.

Planning stage

  1. Define inclusion and exclusion criteria:
    • Language of published reports.
    • Study types. Likely to be all quantitative study types excluding case studies.
    • Specific populations that should be included or excluded (age, risk factors, etc.).
    • Settings (may want to exclude overly specific settings which may induce selection bias).
  2. Decide on the data that should be extracted, e.g., population characteristics, risk estimates, etc.
  3. Create a form that can be used to store extracted data.
  4. Select sources that should be searched. The following should always be included:
    • PubMed
    • Cochrane Central Register of Controlled Trials (CENTRAL)
    • Cochrane Database of Systematic Reviews (CDSR)
    • Web of Science
    • Scopus
    • PsycArticles
    • PsycInfo

Protocol and registration stage

All systematic reviews should be registered in PROSPERO and a protocol based on the items listed under piloting and planning (and PRISMA-P) should be added to the repository prior to review commencement. The form for data extraction should also be added to the repository.

Execution stage

  1. Get a team together consisting of at least 3 people (including the lead).
  2. Search all databases:
    • Create one file for each database (in .bibtex format) and add to the repository.
    • Create one combined file for all datasets with duplicates removed using Zotero (in .bibtex format) and add to the repository.
  3. At least two team members screen all titles and abstracts independently of one another.
    • Create one file per person with pruned articles (in .bibtex format) and add to the repository.
    • Create a combined file of pruned articles with duplicates removed (in .bibtex format) and add to the repository.
  4. At least two team members reads all full-text articles and decides which to include.
    • Create one file per person with articles kept (in .bibtext format) and add to the repository.
    • Create a combined file with all articles kept after discussion (in .bibtext format) and add to the repository.
    • Make notes of exclusion reason at this stage. Add notes to the repository.
    • Make notes of any discussions about inclusion of reports at this stage (disagreements). Add notes to the repository.
    • Find an arbiter if no consensus can be found. Make note of arbiter use and outcome to the repository.

Synthesis and bias assessment

  1. At least two team members use the previously created form to independently extract data from each report. Add all forms to the repository.
  2. Combine extracted data:
    • Compare and discuss differences.
    • Contact authors if necessary.
    • Find an arbiter if no consensus can be found.
    • Make notes of any differences, disagreements, requests from authors, and use of arbiter. Add notes to the repository.
    • Add combined extracted data forms to the repository.
  3. At least two team members independently assesses the risk of bias using ROBINS or ROB (if RCT).
    • Comment on all judgements and add a file per person to the repository.
    • Combine judgements and make note of any discussions regarding disagreements.
    • Find an arbiter if no consensus can be found.
    • Make notes of any differences, disagreements, requests from authors, and use of arbiter. Add notes to the repository.
  4. Create the following and add to the repository:
    • Flow chart for the selection process.
    • Risk of bias matrices/tables.
    • Summary table showing the characteristics of the studies.
    • Summary table which shows the quantitative estimates of the estimands.
  5. Conduct a meta-analysis if suitable and add code and results to the repository.
  6. Assess publication bias using funnel-plots and add to repository.
  7. Use GRADE to assess overall quality of evidence for each estimand and add to repository.

Publication

  1. Find a suitable journal for the systematic review. The journal should fit into the guidelines set by Plan-S and publishing fees should be covered by the Bibsam Agreement.
  2. Write manuscript following PRISMA and journal guidelines.
